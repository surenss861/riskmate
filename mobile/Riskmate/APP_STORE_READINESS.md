# App Store Readiness Guide

## Phase 3: App Store Readiness

### 3.1 Screenshots (Critical)

**Rules:**
- Use dark mode only
- No marketing copy
- No gradients
- No emojis

**Example Captions (Ultra-literal, Apple-approved tone):**
1. "Capture Evidence"
2. "Proof Records"
3. "Verification Details"
4. "Work Records"

**No adjectives. Let the UI sell it.**

**If it sounds like marketing → delete it.**

### 3.2 App Store Description (Tight + Serious)

**Opening line:**
```
RiskMate creates immutable compliance records for field operations.
```

**Then:**
- Evidence capture
- Cryptographic anchoring
- Audit verification
- Offline support

**No hype. Apple loves restraint.**

### 3.3 Privacy & Permissions

**Camera permission string:**
```
"RiskMate needs camera access to capture evidence photos for compliance records."
```

**Photo access:**
```
"Only photos you choose"
```

**No background tracking language.**

This avoids rejection instantly.

### 3.4 Keywords

**Suggested keywords (100 characters max):**
```
compliance,audit,evidence,field operations,safety,construction,immutable,ledger,verification
```

### 3.5 Category

**Primary:** Business
**Secondary:** Productivity

### 3.6 Age Rating

**Content Rating:** 4+ (Business/Productivity)
**No restricted content.**

## Phase 4: Final "Apple Review Pass"

### Before Submission, Verify:

#### Interaction
- ✅ Every tap has feedback
- ✅ No dead taps
- ✅ Sheets dismiss cleanly

#### Performance
- ✅ Cold launch < 1.5s
- ✅ Evidence capture opens instantly
- ✅ Ledger scrolls at 120fps

#### Language
- ✅ No "beta"
- ✅ No "coming soon"
- ✅ No dev jargon exposed

#### Visual
- ✅ No custom alerts
- ✅ No fake UI
- ✅ System sheets everywhere

#### Permissions
- ✅ Camera: Clear, honest reason
- ✅ Photos: "Only photos you choose"
- ✅ No location tracking (unless needed)

#### Privacy Policy
- ✅ Must be accessible
- ✅ Must explain data collection
- ✅ Must explain data usage

## What Apple Will Check

1. **Does it work?** (Functional)
2. **Is it honest?** (No fake features)
3. **Is it safe?** (Privacy, permissions)
4. **Is it polished?** (No crashes, smooth)

You're already 90% there — this is just verification.
